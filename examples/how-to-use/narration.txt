Script to Video. The fastest way to turn a narration script and slides into a polished, narrated video. No video editing software required, just a command line.

The easiest way to start is from a Google Doc. Write your narration in a Google Doc, share it as anyone with the link can view, then run s2v from doc with the URL. The tool pulls the text, splits it into paragraphs, and creates a ready-to-build project. Point it at a folder of screenshots and you are done.

You can also point to a Google Slides presentation or a PowerPoint file instead of individual images. Set slides source to google slides or pptx in your demo dot yaml, and the tool exports each slide as an image automatically. No manual screenshot work needed.

If you prefer the manual route, run s2v init to scaffold a project. This gives you a demo dot yaml config, a narration dot txt template, and a slides folder. Drop your images in, write your narration, one paragraph per slide, and run s2v build. That is the entire workflow.

Now let's talk about voices. Run s2v voices to browse the curated English voices. Ryan Neural is the default British male, great for professional demos. Jenny Neural is an American female, ideal for training. Guy Neural is a casual American male for tutorials. Each voice can be tuned. Set rate to minus ten percent for slower delivery, or plus twenty percent for faster. Adjust pitch with minus four hertz for deeper or plus four hertz for higher.

Slide transitions control how one slide moves to the next. The default is a cross fade, but there are over forty-five options. Wipe left works well for sequential content. Dissolve gives a subtle professional feel. Smooth left feels modern. Circle open creates dramatic reveals. Set it in demo dot yaml or pass dash dash transition on the command line.

Background music is built in. Six presets are generated using audio synthesis, so there are no licence concerns. Ambient is a warm pad for professional demos. Upbeat has rhythmic energy for launches. Corporate is neutral for internal videos. Lo-fi is mellow for tutorials. Set background music to ambient in your config or pass dash dash music ambient on the command line. You can also bring your own music file.

Quality assurance runs automatically before every build. It checks that your slide count matches your narration paragraphs, that your text avoids common speech synthesis problems, and that timings are reasonable. Add custom rules in demo dot yaml to forbid words like to do or fix me in your narration.

Extra features round out the toolkit. Add dash dash subtitles to generate subtitle files synced to your narration. Add dash dash thumbnail to extract a poster frame. Use dash dash watch to auto-rebuild on changes. Run s2v browse to open a local web page where you can preview voices and music before committing to them. And audio is cached, so rebuilds are fast.

For teams using AI coding agents, the tool works as a portable skill. Any agent, whether Copilot, Claude, or Cursor, can use the tool via npx s2v without checking out this repo. Add the skill file to your project and the agent knows all the commands, voices, transitions, and the from doc workflow. Point it at a Google Doc and a folder of screenshots and it builds the video for you.

That is script to video. The simplest path is s2v from doc with a Google Doc URL. The full path is s2v init, write narration, add slides, s2v build. Either way, you get a narrated video in minutes with no video editing needed.
