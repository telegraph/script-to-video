How It Works

1

Write your script

One paragraph per slide in narration.txt. Write naturally, as if presenting to a colleague.

2

Add your slides

Drop PNGs, point to a PowerPoint file, or connect a Google Slides deck.

3

Run s2v build

TTS narration, annotations, transitions, QA checks — all automatic.

Callout Annotations

Highlight regions on slides with golden rectangles and label pills. Perfect for drawing attention to buttons, metrics, or features.

# callouts.json [{ "slide": 2, "label": "Click here", "x": 100, "y": 200, "w": 300, "h": 50 }]

QA Validation

13-point check before every build: slide count, narration quality, callout bounds, timing, and custom forbidden/warned words.

# demo.yaml qa_rules: forbidden_words: ["TODO", "FIXME"] warn_words: ["hack", "workaround"]

Slide Import

Drop PNGs into a folder, or auto-import from PowerPoint or Google Slides.

slides_source: pptx slides_file: presentation.pptx slides_source: google-slides slides_id: 1BxiM...

Extras

Subtitles, thumbnails, watch mode, and TTS caching — all built in.

$ s2v build --subtitles --thumbnail $ s2v build --watch $ s2v browse # local preview UI

Quick Start

# Prerequisites: Node.js 20+, FFmpeg, edge-tts (pip install edge-tts) # Install $ npm install -g script-to-video # Scaffold a new project $ s2v init my-video # Add your slides and edit the narration $ cp ~/slides/*.png my-video/slides/ $ nano my-video/narration.txt # Build $ s2v build my-video # With all the extras $ s2v build my-video --music ambient --subtitles --thumbnail # Explore options $ s2v voices # browse voices $ s2v transitions # browse transitions $ s2v music # browse music presets $ s2v browse # open local preview UI
